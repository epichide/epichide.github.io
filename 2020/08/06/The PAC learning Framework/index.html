CH2 :The PAC Learning Framework

Posted on 2020-08-06 | In math

[TOC] 作者: Mehryar Mogri ## 前言 * PAC学习框架,PAC : probably approximately correct learning ## 中英文释义 definition 定义 note 注解,强调,注意 denote 用什么表示什么 instance 实例 non-trivial ### 符号

\(\mathcal{X}\) : 所有可能的 examples or instances,input space \(\mathcal{Y}\) : 所有可能的 labels or target values,output space c : target concept,类似于映射,从\(\mathcal{X}\) 到 \(\mathcal{Y}\) mapping,是我们想要研究的目标 \(\mathcal{C}\) : concept class ,是一簇我们希望学习的概念.我不知道对不对,可能就是一种对concept的举一反三,扩展模型,没有绝对的边界.比如concept是一个确定角度和边长三角形,concept class就是一个三角形的大概念或者是一个多边形的概念,我们想学习可扩展的concept集合,而不是只用于目前问题的concept 假设assume样例是独立同分布的( independently and identically distributed (i.i.d.))服从于 according to 一个未知分布\(\mathcal{D}\) \(\mathcal{H}\) : 假设集, hypothesis set,是研究者考虑的可能cencept set,它可能会和\(\mathcal{C}\) 一致,也可能不一致,因为我们不知道c和\(\mathcal{C}\)是怎样的,只能靠猜测 \(\mathcal{S}=(x_1,...,x_m)\) : 样本集,来自于\(\mathcal{D}\),同时标注了label \((c(x_1),...,c(x_m))\),标注了的样本集,叫labeled sample,其中每个可以叫做example/instance \(\mathcal{A}\) : 一个算法,algorithm,它依据样本集从\(\mathcal{H}\)中选出一个hs n : 表示represent任意元素x∈\(\mathcal{X}\)所需要的最大计算开销computation cost,比如n维空间需要n个数表示一个元素x size(c) : 表示c的最大计算开销 ## PAC学习模型

  • 泛化误差 generalization error / risk error/ ture error/ simply error 给定假设h∈\(\mathcal{H}\),h的泛化误差定义为:注意这里x~\(\mathcal{D}\) \(R(h)=\underset{x \sim D}{\mathbb{P}}[h(x) \neq c(x)]=\underset{x \sim D}{\mathbb{E}}\left[1_{h(x) \neq c(x)}\right]\)
  • 经验误差 Empirical error/Empirical risk 给定假设h∈\(\mathcal{H}\),h的经验定义为:这里的\(\mathcal{S}\)~\(\mathcal{D^m}\) \(\widehat{R}_{S}(h)=\frac{1}{m} \sum_{i=1}^{m} 1_{h\left(x_{i}\right) \neq c\left(x_{i}\right)}\)
  • 经验误差的期望=泛化误差 \(\underset{S \sim \mathcal{D}^{m}}{\mathbb{E}}\left[\widehat{R}_{S}(h)\right]=R(h)\) proof: \(\underset{S \sim \mathcal{D}^{m}}{\mathbb{E}}\left[\widehat{R}_{S}(h)\right]=\frac{1}{m} \sum_{i=1}^{m} \underset{S \sim \mathbb{D}^{m}}{\mathbb{E}}\left[1_{h\left(x_{i}\right) \neq c\left(x_{i}\right)}\right]=\frac{1}{m} \sum_{i=1}^{m} \underset{S \sim \mathcal{D}^{m}}{\mathbb{E}}\left[1_{h(x) \neq c(x)}\right]=\underset{S \sim \mathcal{D}^{m}}{\mathbb{E}}\left[1_{h(x) \neq c(x)}\right]=\underset{x \sim \mathcal{D}}{\mathbb{E}}\left[1_{h(x) \neq c(x)}\right]=R(h)\) ### PAC learning 如果一个\(\mathcal{C}\),存在一个算法\(\mathcal{A}\)和一个多元多项式polynominal function poly(1/ε, 1/δ, n, size(c)) ,对于?ε>0,δ>0,\(\mathcal{D}\),c,如果满足: \(\underset{S \sim \mathcal{D}^{m}}{\mathbb{P}}\left[R\left(h_{S}\right) \leq \epsilon\right] \geq 1-\delta\) 那么\(\mathcal{C}\)就认为是PAC learnable 再如果?算法\(\mathcal{A}\)在poly(1/ε, 1/δ, n, size(c))时间内运算完,那么那么\(\mathcal{C}\)就认为是efficiently PAC learnable,\(\mathcal{A}\)则称为PAC learning 算法
    • approximately correct (ε)近似准确,即\(h_{S}\)与c之间的误差\(R\left(h_{S}\right) \leq \epsilon\)
    • probability (1-δ) 误差小于epsilon的可能性,即 \(\underset{S \sim \mathcal{D}^{m}}{\mathbb{P}}\left[R\left(h_{S}\right) \leq \epsilon\right] \geq 1-\delta\) #### 说明
    • PAC框架framwork是distribution-free 模型,对于总体分布没有要求
    • 可以放宽以适应domain adaptation 问题
    • PAC框架处理概念集\(\mathcal{C}\)的可学习问题,而不是特定的c.注意，概念类C是算法已知的，但是目标概念c∈C当然是未知的。
    • 当computational representation没有讨论或比较容易得到straightforward,我们可先忽略n和size(c) ### 例子2.4(学习轴对齐的矩形) 考虑平面plane上点集,\(\mathcal{X}=\mathbb{R^2}\),概念集\(\mathcal{C}\)是\(\mathbb{R^2}\)中与坐标轴平行的矩形,R表示目标矩形(c)，R'=\(R_s\)表示假设矩形(\(h_s\))
