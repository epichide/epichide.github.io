符号说明

  1. operation : 在命令的最前面是NIDAQ 工具箱MX XOP(我不清楚XOP是什么的缩写)的操作. 和函数的区别不清楚
  2. function : 是普遍的函数形式,会有返回值
  3. Flags : 以 /sign 的形式, 有时候需要您输入具体的参数值,则为/sign=value的形式
  4. channel : 设备通常有多个输入/输出通道
  5. parameter : 通常是必要参数,是需要完成操作必须指定的内容,通常也有可选部分.相同parameter 之间用;作为分割

默认参数

  1. min_V,max_V: 如果min_V, max_V未指定 默认-10,+10
  2. /EDGE = edge : edge=1,上升向(0为下降相)

DAQmx_AI_SetupReader

设置模拟信号的输入通道,用于接下来读取.

DAQmx_AI_SetupReader /DEV=DeviceNameStr parameterString1 [;parameterString2;parameterString3]

  • flags /DEV=DeviceNameStr : DAQ 设备的名字,例如"dev1"

  • Parameters parameterString : 指定通道号(channel number),同时可以添加通道的配置信息(configuration information),形式为
    parameterString [/type , min_V , max_V];

    • type : 设置信号模式 Diff: 差分,differential
      RSE: 参考单端,reference single ended
      NRSE: 非参考单端, non-reference single ended
      PDIFF: 伪差分, pseudo-differential

      如果没有设置,默认按照设备出厂的默认设置.有关不同设备的默认设置，您需要查阅NI-DAQmx驱动程序随附的NI-DAQmx帮助。并非所有设备都支持所有类型。

      关于信号模式的解释和选择,请参考DIFF,RSE,NRSE,PDIFF信号模式

    • min_V,max_V

DAQmx_Scan

执行输入扫描操作(scan operation),获取数据,存到igor wave 或者 igor FIFO中,您必须WAVES和FIFO选其一,异或选择关系.
Wave和FIFO通道可以是不超过8bit的任何signed的数字类型。不允许unsigned，也不允许复数的。所有波形或FIFO通道必须具有相同的数字类型。 如果使用整数的波或FIFO通道，则以原始模式(raw mode)读取数据。也就是说，数据是unscaled的 A / D 转换器计数。 如果使用浮点的波或FIFO通道，则数据将采用scaled电压的形式。 对于12位DAQ设备，该值的范围是-4096至+4095； 16位设备将产生从-32768到+32767的值。 (不确定和前面的8bit什么关系) 不允许使用/ AVE标志对整数求平均

DAQmx_Scan /DEV=DeviceNameStr [/AVE=nAverage /BKG[=doBackground] /CLK={clockSpec} /ERRH=ErrorHookStr /EOSH=EndOfScanHookStr /PAUS={pauseTriggerSpec} /RPT[=repeatSpec] /RPTC[=repeatSpec] /RPTH=RepeatHookStr /RTRG={refTrigSpec} /SINT=sampleInterval /STRT[=startSpec] /TRIG={trigspec} [WAVES=WaveParameterString] [FIFO=FIFOParameterString]

  • waves 对于长度有限的扫描读取操作(scanning operation)
    使用 WAVES = WaveParameterString , 获取的数据将存储在您命名(WaveParameterString)的waves中. 数的大小由waves 设定的长度决定; 您必须确定号要获取多少数据
    如果多个通道被采集,您必须提供多个波名,以供存储.所有波是统一的长度.采样周期由WaveParameterString中第一个波的wave scaling的dX因子设置. 详见: Analog Input: Waves and Wave Scaling.(没有找到链接)

  • FIFO

    对于不确定长度的扫描操作，可以选择将数据流传输到硬盘。
    使用FIFO = FIFOParameterString会使获取的数据存储在参数字符串中命名的Igor FIFO中。连续获取数据，直到使用fDAQmx_ScanStop（）取消操作为止。
    开始采集后，控制权立即返回Igor Pro。 / BKG标志被忽略。原始数据的实际采集是通过后台处理进行的。其他Igor操作，包括将获取的数据的一部分传输到wave，可以在获取数据时进行。 FIFO抯还可以在采集过程中自动将数据存储在磁盘文件中
    扫描速率由您提供的用于接收数据的FIFO的delta T参数控制。请参见《 NIDAQ工具MX指导》 [连续模拟输入到FIFO中]和《 NIDAQ XOP：程序员介绍》 [模拟输入：使用FIFO中]。

  • flag

    /DEV = DeviceNameStr : 字符串,DAQ设备名

    /AVG = nAverage : 使扫描操作获取nAverage连续读数，并对返回的每个数据点取平均值。请注意，这会将实际扫描速率提高了nAverage倍。比如我要获取n个点,实际采集了n*nAverage个点.

    /BKG = doBackground : 如果doBackground为1，则在后台进行传输。如果doBackground为0，则DAQmx_Scan在获取所有数据之前不返回（/ STRT = 0除外，请参见下文）。如果/ BKG标志不存在，则默认/ BKG = 0。

    /CLK = {srcStr,edge} : 1)指定默认时钟以外的采样时钟。您可以将自己的时钟连接到外部引脚，也可以从内部信号（例如计数器的输出）中获取采样时钟。字符串 srcStr 命名时钟信号的源端子。通常，这类似于/ devname / pfi0或/ devname / ctr0 内部输出，其中“ devname”与DeviceNameStr的字符串相同。2)时钟脉冲触发扫描的部分由 edge 设置。0：下降沿; 1：上升沿. 如果您使用自己的时钟源，则由波形缩放或FIFO设置的扫描速率将被忽略，但NI-DAQmx驱动程序可以使用它来优化扫描操作。

    /ERRH = ErrorHookStr : 如果扫描操作期间发生错误，则包含要执行的命令的字符串。仅在后台扫描期间使用（请参阅/ BKG标志）。

    /EOSH = EndOfScanHookStr : 扫描完成时要执行的命令的字符串。仅在后台扫描期间使用（请参阅/ BKG标志）。

    /PAUS={source, pretrigSamples [, type, edgeslopewhen, level1, level2]} : 设置一个暂停触发器，允许硬件控制扫描操作。就是说，在asserte暂停触发器时暂停扫描，并在未assert暂停触发器时继续从上次中断的地方继续扫描。
    称它为触发器有点误导人. 因为按照数字逻辑来讲,触发器通常与延边(edge)变化有关.在这里的暂停触发器是由电平(level)控制的,而不是边(edge).门(gate)或许是一个更合适的描述.

    • souce : 字符串,指定终端源,通常是PFI 输入端.类似于/ devname / pfi1，其中“ devname”与DeviceNameStr使用的字符串相同。没有默认源.

    • type : 数字，指定要使用的触发器类型: 0: Disable (不设置暂停触发器 pause trigger)
      1: Digital 数字
      2: Analog level 模拟信号电平
      3: Analog window
      Default is type 1.

    • edgeslopewhen : 控制何时暂停扫描操作. 值有: 0: low level (digital), below level1 (analog level) or outside the window (analog window)
      1: high level, above level1, or inside window.
      Default is 1.

    • level1 : 模拟电平触发的电压阈值，或模拟窗口底部的电压。忽略数字触发。默认为0

    • level2 : 在模拟窗口的顶部设置电压。忽略数字触发或模拟电平触发。默认值为0.。什么是模拟窗口 analog window

    /RPT : (仅适用于WAVES关键字）导致在获取所有数据时重新启动扫描操作。再次使用相同的波形，覆盖先前获取的数据。这将导致示波器类型的模式。一次扫描的结束与下一次扫描的开始之间可能会有一些延迟，因为仅在Igor的主偶数循环或调用DoXOPIdle时才检测到扫描的结束。如果扫描操作使用触发器，则每次扫描都使用相同的触发器。注意：这与以前的NIDAQ Tools版本不同。不太清楚偶数循环时什么意思.该标志不适用于获取FIFO。

    /RPTC : （仅适用于WAVES关键字）样本将连续采集到波形中。当波被填充时，下一个样本进入波的第零行。如果扫描操作使用触发器，则仅触发第一次扫描。随后的扫描与第一次扫描连续进行。此模式消除了使用/ RPT时两次扫描之间存在的等待时间，但代价是第一次扫描时不使用触发器。此行为与旧版NIDAQ Tools软件包中的fNIDAQ_ScanWavesRepeat相同。该标志不适用于获取FIFO。

    /RPT
    /PRTC = repeatSpec : 如果repeatSpec为1，就像/ RPT或/ RPTC一样。如果repeatSpec为0，就像省略/ RPT一样。

    /RPTH = RepeatHookStr : 字符串命令，重复扫描（请参阅/ RPT标志）完成一次扫描时要执行的命令。请注意，hook命令将在一次扫描完成之后且在下一次扫描开始之前执行。因此，该命令将延迟下一次重复扫描的开始。

    /RTRG={source, pretrigSamples [, type, edgeslopewhen, level1, level2]} : 设置参考或停止触发，以进行预触发采集(pre-trigger acquisition).就是说，扫描持续进行直到触发被assert为止。在那一点上，从已经获取的样本中传输pretrigSamples数据点，并继续扫描直到获取（totalSamples-pretrigSamples）个样本.在采集所有样本之前，NI-DAQmx将不允许任何数据传输。后台扫描仍然有效，但是在扫描期间您不会看到wave更新新数据。不能与连续获取到FIFO一起使用。 source是给出触发信号源端子的字符串。通常，这将是/ devname / pfi1，其中“ devname”是与DeviceNameStr相同的字符串。如果将“”用作信号源，则将使用默认的停止触发输入，通常是PFI1或DAQ设备的I / O连接器上的引脚10。pretrigSamples是从参考触发器到达之前从获取的数据中保存的样本数。

    • type 是一个数字，指定要使用的触发器类型。值有:: 0: Disable triggers (use software start)
      1: Digital
      2: Analog level
      3: Analog window
      Default is type 1.
    • edgeslopewhen 触发用于数字触发的触发脉冲部分，或模拟电平的斜率，或模拟窗口的窗口感。值是：0：下降沿（数字），下降斜率（模拟水平）或离开窗口（模拟窗口）.1：上升沿，上升斜率或进入窗口。
    • level1 : 模拟电平触发的电压阈值，或模拟窗口底部的电压。忽略数字触发。默认为0
    • level2 : 在模拟窗口的顶部设置电压。忽略数字触发或模拟电平触发。默认值为0.。什么是模拟窗口 analog window

    /SINT=sampleInterval : 设置一次扫描中连续通道之间的时间间隔，而与一个通道上样本之间的时间间隔（扫描间隔）无关。即，扫描间隔可以是0.01秒，并且采样间隔可以是0.0001秒以近似同时采样。例如：
    DAQmx_Scan /SINT=.0001 wave =“ input1，1; input2，2”
    将采样通道1和2，将数据存储在wave input1和input2中。采样通道1和通道2之间的时间段将设置为0.0001秒。
    将sampleInterval设置为零将选择默认值，通常是扫描间隔除以通道数。
    注意：“采样间隔”和“扫描间隔”是NI在其NI-DAQ Traditional文档中使用的短语。他们现在使用术语“ AI转换率”和“采样率”。

    /STRT = startSpec : 控制是否立即开始扫描操作。如果指定了触发器，则启动与触发器不同。在开始操作之前，即使施加了触发信号也没有任何反应。操作开始后，如果使用/ TRIG或/ RTRG指定了触发，则扫描将一直等到施加触发信号。如果startSpec为零，则不会自动开始扫描操作。在这种情况下，请使用fDAQmx_ScanStart（）函数开始扫描。如果startSpec不为零，则在AQmx_Scan将控制权返回给Igor之前开始扫描操作。如果未使用/ STRT标志，则它类似于/ STRT = 1。

    / STRT : 在没有startSpec的情况下，/ STRT标志的行为类似于/ STRT = 1。

    /TRIG={source [, type, edgeslopewhen, level1, level2]} : 设置启动触发器。扫描在开始任何扫描之前等待指定的信号。请参阅/ RTRG标志以获取有关source, type, edgeslopewhen, level1 and level2.

  • detail

    什么时候应该使用/ RPT或/ RPTC？

    这些标志要求使用示波器类型的模式，在该模式下，波形会重复填充数据。使用图形来监视某些过程或重复波形非常有用。

    使用/ RPT时，每一次扫描都在最后一次扫描结束后立即开始。它通过停止采集任务，执行重复挂接（/ RPTH，如果有）然后重新启动任务来完成此任务。因为对于每个新的扫描都会重新启动任务，所以如果配置了触发器，则每次扫描都会等待触发信号。如果您有合适的触发信号，这将使扫描与您正在监视的过程保持同步。如果尚未配置触发器，则下一次扫描相对于上一次扫描的结束以未知的延迟开始。此延迟将取决于计算机的速度，执行重复挂接所花费的时间，计算机上可能有多少其他活动以及其他许多因素。

    使用/ RPTC时，每次扫描只是连续采集中的下N个点。采集任务是连续的，该任务不会停止并重新启动。因此，在扫描之间不存在与/ RPT相同的时序问题。但是，如果已配置触发器，则该触发器仅用于启动操作。第一次扫描之后，仅以指定的扫描速率继续扫描

    当您要监视连续过程时，请使用/ RPTC，在该过程中波形以可预测的方式连续不断地重复，并且没有合适的触发信号可用于将扫描与要监视的信号的开始同步。

DAQmx_AO_SetOutputs

配置模拟信号的输出通道,并指定输出的电压或电流(如果您的DAQ设备支持的话).一次调用设置所有输出.

DAQmx_AO_SetOutputs /DEV=DeviceNameStr [/CRNT=[currentMode]] [/KEEP=[keepTask]] parameterString[;parameterString2;parameterString3]

  • flag

    /DEV=DeviceNameStr : DAQ 设备的名字,例如"dev1"
    /CRNT[=currentMode] : 如果您的设备支持电流输出,可以使用该flag, 之后指定的min_V和max_V 单位都是安培(A,Amps)
    /KEEP=[=keepTask] : 与任务是否需要destroy相关. 当该flag存在,且keepTask不为0,或者未指定(absent),或者不存在,则操作完成返回igor后,创建的任务会持续存在,不被destroy.

  • parameter

    parameterString : 指定channel,范围从0~您设备的最大通道数.加上额外的最大/最小值信息,是电min_V压还是电流取决于/CRNT的设置.形式为:
    outputSetting, chnnel[,min_V,max_V]

  • example

  • detail

    任务创建和销毁是很耗费时间点的.如果您使/KEEP=1,持久不关闭,与每次输出创建又销毁相比会快500倍.如果您在初始调用DAQmx_AO_SetOutputs 后,使用 fDAQmx_AO_UpdateOutputs(#fDAQmx_AO_UpdateOutputs) 设定模拟信号输出则会更快.

    /KEEP=1的坏处是它会占用模拟信号的输出 , 以至于在占用期间不能以其它方式使用.在您使用该函数期间,您不能使用其它任何输出.

    当您要销毁它,有两种方式:

    1. 再次调用该函数,但令/KEEP=0,或不使用/KEEP flag. 这个函数可以让您对DAQ输出最后一次信息后关闭(使得您可以设定一些状态信息)
    2. 调用 fDAQmx_WaveformStop.

DAQmx_CTR_CountEdges

使用板载计数器/计时器(onboard counter/timer)设置并可选地开始边沿计数操作。 使用fDAQmx_CTR_ReadCounter（）或fDAQmx_CTR_ReadWithOptions（）检索当前计数。

DAQmx_CTR_CountEdges /DEV=DeviceNameStr [/EDGE=edge] [/INIT=initialCount] [/DIR=direction] [/CLK=clkSource [/FIFO=fifoName] [/WAVE=waveName /RPT]] /ERRH=ErrorHookStr /EOSH=EndOfScanHookStr /RPTH=RepeatHookStr [/STRT[=startSpec]] [/SRC=sourceTerminal] [/TRIG={trigspec}] [/PAUS={pauseTriggerSpec}] [/OUT=outputTerminal] counter

  • flag

    /DEV=DeviceNameStr : DAQ 设备名
    /EDGE=edge : 决定计数那种类型的边,0:下降,1:上升
    /INIT=initialCount : 预设计数器内容初始值,默认为0
    /DIR=direction : 设定计数方向,direction是+1,还是-1;如果为0,则方向由外部信号控制,如下表. 我不知道这里的pin引脚要怎么弄. 这些是通道或者位bit的符号,详见

    DAQ device counter 0 counter 1
    M-series PFI 10 PFI 11
    pin 45 pin 46
    E- and S-series port0/line6 port0/line7
    pin 16 pin 48

    /ERRH = ErrorHookStr : 后台计数到wave或FIFO(first in first out 先进先出)期间发生错误时要执行的命令,字符串形式
    /EOSH= EndOfScanHookStr : 在后台计数完成后要执行的命令,字符串形式.因为FIFO操作永远不会停止,所以仅适用于/WAVE flag.
    /FIFO=fifoName : 用于接收多个读数的Igor FIFO的名称。如果使用/ FIFO，则还必须使用/ CLK指定时钟脉冲源. 该操作不使用板载时基：您必须使用/ CLK标志提供时钟脉冲源。因为使用了外部时钟源，所以由FIFO的deltat设置设置的周期不会设置采样率。但是，采样率仍会传递给NI-DAQmx驱动程序，该驱动程序可使用该信息来优化计数器设置。您应该将deltat设置为期望的最大速率。使用FIFO意味着可以无限期地获取计数器读数

Error Handling

fDAQmx_ErrorString

在执行NIDAQ Tools MX操作或函数期间或在操作的后台处理期间发生错误时，错误信息将存储在堆栈中。此函数以字符串形式返回最近错误的信息。然后，它将信息从堆栈中删除。 后续调用将返回来自先前任何错误的信息，依此类推。堆栈仅限于保留有关最后五个错误的信息。返回的信息将包括错误号和描述性消息。如果发生NI-DAQmx驱动程序错误，该消息可能会很长。这些很长的消息包含非常有用的信息。

fDAQmx_ErrorString()

  • return : 字符串,错误信息。如果没有错误信息可用，则返回一个空字符串。
