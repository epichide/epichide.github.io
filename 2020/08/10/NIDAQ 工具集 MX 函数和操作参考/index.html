符号说明

  1. operation : 在命令的最前面是NIDAQ 工具箱MX XOP(我不清楚XOP是什么的缩写)的操作. 和函数的区别不清楚
  2. function : 是普遍的函数形式,会有返回值
  3. Flags : 以 /sign 的形式, 有时候需要您输入具体的参数值,则为/sign=value的形式
  4. channel : 设备通常有多个输入/输出通道
  5. parameter : 通常是必要参数,是需要完成操作必须指定的内容,通常也有可选部分.相同parameter 之间用;作为分割

默认参数

  1. min_V,max_V: 如果min_V, max_V未指定 默认-10,+10
  2. /EDGE = edge : edge=1,上升向(0为下降相)

函数

DAQmx_AI_SetupReader

设置模拟信号的输入通道,用于接下来读取.

DAQmx_AI_SetupReader /DEV=DeviceNameStr parameterString1 [;parameterString2;parameterString3]

  • flags /DEV=DeviceNameStr : DAQ 设备的名字,例如"dev1"

  • Parameters parameterString : 指定通道号(channel number),同时可以添加通道的配置信息(configuration information),形式为
    parameterString [/type , min_V , max_V];

    • type : 设置信号模式 Diff: 差分,differential
      RSE: 参考单端,reference single ended
      NRSE: 非参考单端, non-reference single ended
      PDIFF: 伪差分, pseudo-differential

      如果没有设置,默认按照设备出厂的默认设置.有关不同设备的默认设置，您需要查阅NI-DAQmx驱动程序随附的NI-DAQmx帮助。并非所有设备都支持所有类型。

      关于信号模式的解释和选择,请参考DIFF,RSE,NRSE,PDIFF信号模式

    • min_V,max_V

      (我不确定,参考)

  • example

  • detail

    缺点是DAQmx_AI_SetupReader创建了一个模拟输入任务，该任务占用了模拟输入，因此不能以任何其他方式使用它们。如果通过此操作使用模拟输入的子集，则无法对其他输出执行任何操作，直到使用fDAQmx_ScanStop销毁任务为止

DAQmx_AO_SetOutputs

配置模拟信号的输出通道,并指定输出的电压或电流(如果您的DAQ设备支持的话).一次调用设置所有输出.

DAQmx_AO_SetOutputs /DEV=DeviceNameStr [/CRNT=[currentMode]] [/KEEP=[keepTask]] parameterString[;parameterString2;parameterString3]

  • flag

    /DEV=DeviceNameStr : DAQ 设备的名字,例如"dev1"
    /CRNT[=currentMode] : 如果您的设备支持电流输出,可以使用该flag, 之后指定的min_V和max_V 单位都是安培(A,Amps)
    /KEEP=[=keepTask] : 与任务是否需要destroy相关. 当该flag存在,且keepTask不为0,或者未指定(absent),或者不存在,则操作完成返回igor后,创建的任务会持续存在,不被destroy.

  • parameter

    parameterString : 指定channel,范围从0~您设备的最大通道数.加上额外的最大/最小值信息,是电min_V压还是电流取决于/CRNT的设置.形式为:
    outputSetting, chnnel[,min_V,max_V]

  • example

  • detail

    任务创建和销毁是很耗费时间点的.如果您使/KEEP=1,持久不关闭,与每次输出创建又销毁相比会快500倍.如果您在初始调用DAQmx_AO_SetOutputs 后,使用 fDAQmx_AO_UpdateOutputs(#fDAQmx_AO_UpdateOutputs) 设定模拟信号输出则会更快.

    /KEEP=1的坏处是它会占用模拟信号的输出 , 以至于在占用期间不能以其它方式使用.在您使用该函数期间,您不能使用其它任何输出.

    当您要销毁它,有两种方式:

    1. 再次调用该函数,但令/KEEP=0,或不使用/KEEP flag. 这个函数可以让您对DAQ输出最后一次信息后关闭(使得您可以设定一些状态信息)
    2. 调用 fDAQmx_WaveformStop.

fDAQmx_WriteChan()

写入电压一次,到指定的模拟输出通道。指定的电压应立即出现在指定的模拟输出上。

fDAQmx_WriteChan(Device, Channel, Volts, Min_V, Max_V )

  • parame

    device : 字符串,设备名
    channel : 数值,要读取的模拟输出通道号( 不清楚为什么一定要读取)。范围是0到n-1，其中n是设备上模拟输入通道的数量。
    Volts : 要输出的电压
    Min_V : 数值,最小输出范围
    Max_V : 数值,最大输出值

  • return : 0为成功

  • detail NI-DAQmx驱动程序将不接受Min_V = Max_V，也不会接受设备可实现范围之外的Min_V或Max_V。 fDAQmx_ErrorString（）返回的错误消息将告诉您可接受的限制。

    任务的执行过程: 通过创建任务，向任务添加指定的通道，向任务写入值，最后停止并清除任务

    是否表明它很慢

fDAQmx_WaveformStop()

使用该函数停止waveform产生(generate,是output). 您必须使用此功能来停止设置为无限期继续的波形生成。

fDAQmx_WaveformStop(Device)

  • parameter

    device : 设备名,字符串

  • return

    0 为 succeed, non zero 为 error

DAQmx_CTR_CountEdges

使用板载计数器/计时器(onboard counter/timer)设置并可选地开始边沿计数操作。 使用fDAQmx_CTR_ReadCounter（）或fDAQmx_CTR_ReadWithOptions（）检索当前计数。

DAQmx_CTR_CountEdges /DEV=DeviceNameStr [/EDGE=edge] [/INIT=initialCount] [/DIR=direction] [/CLK=clkSource [/FIFO=fifoName] [/WAVE=waveName /RPT]] /ERRH=ErrorHookStr /EOSH=EndOfScanHookStr /RPTH=RepeatHookStr [/STRT[=startSpec]] [/SRC=sourceTerminal] [/TRIG={trigspec}] [/PAUS={pauseTriggerSpec}] [/OUT=outputTerminal] counter

  • flag

    /DEV=DeviceNameStr : DAQ 设备名
    /EDGE=edge : 决定计数那种类型的边,0:下降,1:上升
    /INIT=initialCount : 预设计数器内容初始值,默认为0
    /DIR=direction : 设定计数方向,direction是+1,还是-1;如果为0,则方向由外部信号控制,如下表. 我不知道这里的pin引脚要怎么弄. 这些是通道或者位bit的符号,详见

    DAQ device counter 0 counter 1
    M-series PFI 10 PFI 11
    pin 45 pin 46
    E- and S-series port0/line6 port0/line7
    pin 16 pin 48

    /ERRH = ErrorHookStr : 后台计数到wave或FIFO(first in first out 先进先出)期间发生错误时要执行的命令,字符串形式
    /EOSH= EndOfScanHookStr : 在后台计数完成后要执行的命令,字符串形式.因为FIFO操作永远不会停止,所以仅适用于/WAVE flag.
    /FIFO=fifoName : 用于接收多个读数的Igor FIFO的名称。如果使用/ FIFO，则还必须使用/ CLK指定时钟脉冲源. 该操作不使用板载时基：您必须使用/ CLK标志提供时钟脉冲源。因为使用了外部时钟源，所以由FIFO的deltat设置设置的周期不会设置采样率。但是，采样率仍会传递给NI-DAQmx驱动程序，该驱动程序可使用该信息来优化计数器设置。您应该将deltat设置为期望的最大速率。使用FIFO意味着可以无限期地获取计数器读数

fDAQmx_ScanStop()

使用此功能可在完成背景扫描操作之前将其停止。这将释放扫描通道以进行其他操作。调用此函数时不会传输任何数据

fDAQmx_ScanStop(Device)

  • parameter device : 字符串,DAQ设备名

fDAQmx_ResetDevice()

对DAQ设备进行硬件复位。结果，该设备上的所有操作将停止(cease)。 参数Device给出DAQ设备名称的字符串。

fDAQmx_ResetDevice（Device) * parameter

device  :   字符串,设备名
  • return : 0 成功

fDAQmx_ErrorString()

在执行NIDAQ Tools MX操作或函数期间或在操作的后台处理期间发生错误时，错误信息将存储在堆栈中。此函数以字符串形式返回最近错误的信息。然后，它将信息从堆栈中删除。 后续调用将返回来自先前任何错误的信息，依此类推。堆栈仅限于保留有关最后五个错误的信息。返回的信息将包括错误号和描述性消息。如果发生NI-DAQmx驱动程序错误，该消息可能会很长。这些很长的消息包含非常有用的信息。

fDAQmx_ErrorString()

  • return : 字符串,错误信息。如果没有错误信息可用，则返回一个空字符串。
